at_c

atProtocol implementation in C for ESP32 and Embedded Devices

C CMake Git/GitHub Bash Scripting GitHub Actions - CI/CD Microcontrollers (ESP32) Cryptography MbedTLS Embedded Systems Socket Programming

In-Progress

at_c is the client implementation of the atProtocol. Similar to libraries needed for HTTP communication, the C SDK is required for embedded devices and C applications to interact with their atServer through the atProtocol. This SDK facilitates edge encryption, server authentication, and simplifies protocol interactions for users. Among the SDKs I contributed to, including at_java, at_pico_w, and at_esp32, at_c emerged as the most complete due to customer demand. By August 2024, we achieved the v0.1.0 beta release.

Cryptography Features:

  • RSA 2048 encrypt/decrypt
  • RSA 2048 sign/verify
  • AES 256 CTR encrypt/decrypt
  • AES key generation
  • RSA 2048 key importing
  • SHA 256 hashing

atProtocol Features:

  • CRUD operations: Put, Get, Delete
  • Events: Notify, Monitor
  • AtKey: metadata, to/from string
  • Authentication: PKAM authentication

GitHub Source Code.

noports

Establish TCP connections without opening ports

C CMake Bash Scripting GitHub Actions - CI/CD Cryptography Docker Networking

In-Progress

noports is a company-wide project designed to allow users to remotely access their machines without requiring open ports. It provides a secure and user-friendly solution, especially for non-tech-savvy users. The project is currently under development and not yet publicly available.

My Contribution:

  • Developed the at_c C SDK, a core dependency for noports' C SSHNPD.
  • Created the initial set of end-to-end tests using Docker to simulate the noports handshake process. This involved closing ports on Docker machines and simulating one container SSH'ing into another.
