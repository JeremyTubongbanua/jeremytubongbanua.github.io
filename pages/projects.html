at_c

In-Progress

atProtocol implementation in C for ESP32 and Embedded Devices

at_c is the client implementation of the atProtocol. Similar to libraries needed for HTTP communication, the C SDK is required for embedded devices and C applications to interact with their atServer through the atProtocol. This SDK facilitates edge encryption, server authentication, and simplifies protocol interactions for users. Among the SDKs I contributed to, including at_java, at_pico_w, and at_esp32, at_c emerged as the most complete due to customer demand. By August 2024, we achieved the v0.1.0 beta release.

Cryptography Features:

  • RSA 2048 encrypt/decrypt
  • RSA 2048 sign/verify
  • AES 256 CTR encrypt/decrypt
  • AES key generation
  • RSA 2048 key importing
  • SHA 256 hashing

atProtocol Features:

  • CRUD operations: Put, Get, Delete
  • Events: Notify, Monitor
  • AtKey: metadata, to/from string
  • Authentication: PKAM authentication

GitHub Source Code.

Qt/Atsign Plant

January 2024

CES 2024 Booth Demo

This was a joint demo with the Qt Company showcased at CES 2024 in Las Vegas. I was flown by Atsign to set up the demo at Qt's booth, where I presented it to attendees and executives. The demo featured a secure IoT plant monitoring system using atProtocol and Qt.

at_esp32

April 2023

atProtocol implementation in C++ for ESP32 Arduino

The at_esp32 SDK enables ESP32 Arduino devices to communicate with their atServers using edge encryption. I implemented various low-level cryptographic functions required by the atProtocol, such as RSA 2048 encrypt/decrypt, AES 256 CTR encrypt/decrypt, and AtClient operations like put, get, delete, and PKAM authentication.

UMass Boston Winter 2022: This SDK was developed to support UMass Boston students in their final CS project, which involved creating an actuator/sensor ESP32 demo using the atPlatform. I conducted workshops, held office hours, and created a video tutorial playlist to guide students in using the SDK successfully.
